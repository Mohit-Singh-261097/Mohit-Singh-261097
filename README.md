### [Customer Segmentation — Olist E-Commerce](https://github.com/Mohit-Singh-261097/Customer-Segmentation-Olist)

![Python](https://img.shields.io/badge/Python-0d0d0d?style=flat&logo=python&logoColor=5DCAA5) ![Scikit-learn](https://img.shields.io/badge/Scikit--learn-0d0d0d?style=flat&logo=scikit-learn&logoColor=5DCAA5) ![Power BI](https://img.shields.io/badge/Power_BI-0d0d0d?style=flat&logo=powerbi&logoColor=5DCAA5)

Segmented 93K+ customers across Brazil using K-Means (k=4), validated against Hierarchical and DBSCAN. The most useful output wasn't the clusters — it was pinning one segment's low review scores (avg 1.6) directly to 19-day average shipping delays. That's a logistics problem dressed up as a satisfaction problem.

|---|---|
| Customers segmented | 93,000+ |
| Segments | 4 |
| Risk segment avg review | 1.6 ★ |
| Avg shipping delay (risk group) | 19 days |

### [Employee Absenteeism Analysis](https://github.com/Mohit-Singh-261097/employee-absenteeism-analysis)

![Python](https://img.shields.io/badge/Python-0d0d0d?style=flat&logo=python&logoColor=5DCAA5) ![Scikit-learn](https://img.shields.io/badge/Scikit--learn-0d0d0d?style=flat&logo=scikit-learn&logoColor=5DCAA5)

Built an absenteeism prediction pipeline on 20K employee records — 85% classification accuracy across 8 models, R² of 0.51 on hours absent. The model correctly flagged 78% of employees who went on to take 20+ absence days. HR gets an early-warning list, not a post-hoc report.

| Metric | Value |
|---|---|
| Records | 20,000 |
| Classification accuracy | 85% |
| Regression R² | 0.51 |
| High-risk employees flagged | 78% |
